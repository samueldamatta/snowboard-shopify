# Auto-tag California customers as high-value — Shopify Flow

**Audience:** a Shopify **Plus** merchant who wants customers who buy from **California** to be
automatically tagged as **high-value**.

**TL;DR:** This is built with **Shopify Flow** — free on Plus, **no theme code**. Flow watches for new
orders, checks whether the order ships to California, and adds a tag to the customer. Everything below
is an **admin automation**; this repo holds the documentation and the exported workflow so the
configuration is version-controlled and reproducible.

---

## Why Shopify Flow (and not theme code)

- **Free for Plus** — Flow is included with the Plus plan; no app subscription.
- **Tagging is a back-office concern** — it has nothing to do with how the storefront renders, so it
  doesn't belong in the Liquid theme. Putting it in Flow keeps the theme clean and the logic in the
  one place merchants and ops can see and edit.
- **Auditable & testable** — Flow has a built-in run log and a "test/run" facility, so we can prove a
  California order produced the tag.

---

## Workflow logic

| Step | Type | Configuration |
|---|---|---|
| 1 | **Trigger** | **Order created** *(default)* — fires when any order is created. Switch to **Order paid** if the tag should only apply after successful payment. |
| 2 | **Condition** | Shipping address province is California. Default check: `order.shippingAddress.provinceCode` **is equal to** `CA`. (Equivalent: `order.shippingAddress.province` equals `California`.) |
| 3 | **Action** | **Add customer tags** → `High Value Customer`. The action targets `order.customer`. |

```
[ Order created ]
        │
        ▼
[ order.shippingAddress.provinceCode  ==  "CA" ]
        │ true
        ▼
[ Add customer tags → "High Value Customer" ]
```

> **Province code vs name:** `provinceCode` (`CA`) is the safest comparison — it's the stable ISO
> subdivision code and is immune to casing/localization differences in the human-readable
> `province` ("California"). Use `provinceCode == "CA"`.

### Optional refinement — gate on spend as well

If "high value" should also depend on order value, add a **second condition** combined with the
province check (Flow AND):

```
provinceCode == "CA"
        AND
order.currentTotalPriceSet.shopMoney.amount  >  250    # threshold to confirm with client
```

Only orders that ship to California **and** exceed the spend threshold then get the tag.

## Build it in the admin (authoritative steps)

The canonical artifact is produced from the Flow editor; these steps create it:

1. **Shopify admin → Flow → Create workflow.**
2. **Trigger:** search **Order created** (Shopify trigger) and select it.
   - *Alternative:* **Order paid**, if tagging should wait for payment.
3. **Add a Condition** step, connected to the trigger's output:
   - Set the field to **Order → Shipping address → Province code**
     (`order.shippingAddress.provinceCode`).
   - Comparator **is equal to**, value **`CA`**.
   - *(Optional)* add a second criterion **Order → Current total price → Shop money → Amount**
     **is greater than** your threshold; keep the logic as **AND**.
4. **Add an Action** on the condition's **true** branch:
   - Choose **Add customer tags** (Shopify action).
   - Tags: **`High Value Customer`**.
5. **Name** the workflow `Tag California customers as high value`.
6. **Turn it on.**
7. **Export for the repo:** open the workflow → **⋯ (More actions) → Export**. Shopify downloads a
   `.flow` file (JSON). Commit it next to this doc as
   [`docs/flow-california-high-value.flow`](./flow-california-high-value.flow), and add an admin
   screenshot of the workflow graph as `docs/flow-california-high-value.png`.

> The exported `.flow` is the source of truth — it can be re-imported via **Flow → Import** to recreate
> the workflow on another store/environment.

## Representative export (illustrative)

The block below shows the workflow's logic in the shape of a Flow export, **annotated for review**.
Treat it as documentation of intent — the real, importable artifact is the `.flow` file you download
from **Export** in step 7 above (its exact field IDs/handles are generated by Flow at export time).

```jsonc
{
  "name": "Tag California customers as high value",
  "trigger": {
    "handle": "shopify::order::created",          // swap to "order::paid" to gate on payment
    "outputs": ["order"]
  },
  "steps": [
    {
      "id": "condition-province",
      "type": "condition",
      "logic": "AND",
      "criteria": [
        {
          "field": "order.shippingAddress.provinceCode",
          "operator": "==",
          "value": "CA"
        }
        // optional spend gate:
        // { "field": "order.currentTotalPriceSet.shopMoney.amount", "operator": ">", "value": "250" }
      ]
    },
    {
      "id": "action-tag-customer",
      "type": "action",
      "handle": "shopify::customer::addTags",
      "branch": "condition-province.true",
      "inputs": {
        "customerId": "{{order.customer.id}}",
        "tags": ["High Value Customer"]
      }
    }
  ]
}
```

## Open items (confirm with client)

These are decisions the client owns. Defaults used above are marked, and are easy to flip in the Flow
editor:

| Question | Default used | Alternatives |
|---|---|---|
| Exact tag string | `High Value Customer` | `VIP`, `California VIP`, or apply **both** |
| Trigger timing | **Order created** | **Order paid** (only after payment clears) |
| Address source | **Shipping** address province | **Billing** address province (`order.billingAddress.provinceCode`) |
| Spend gate | none (province only) | add amount threshold, e.g. `> 250` |

---

## Verification

1. In Flow, open the workflow and use **Test/Run** against a **California** test order — or place a real
   test order with a California shipping address.
2. Confirm the run appears in the workflow's **run log** as completed (condition met → action ran).
3. Open the order's **customer** record in the admin and confirm the **`High Value Customer`** tag is
   present.
4. **Negative check:** a non-California order should show the run **stopping at the condition** with no
   tag added.
